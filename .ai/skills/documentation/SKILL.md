# Documentation Skill

## Use this when

- setup changed
- architecture changed
- roadmap changed
- feature behavior changed
- AI behavior changed
- deployment process changed
- a contract, migration, or rollout expectation changed

## Documentation checklist

- accurate
- concise
- current
- easy to scan
- useful for future self
- useful for interview/demo review

## ADR triggers

Write or update an ADR when the change affects:

- module boundaries
- transaction or consistency strategy
- external contracts
- AI safety posture
- rollout or rollback behavior
