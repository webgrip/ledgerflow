# Product Planning Skill

## Use this when

- planning a new feature
- defining a milestone
- writing a feature brief
- deciding what to build next
- cutting scope
- deciding how to slice a risky change safely

## Process

1. Identify the current roadmap phase.
2. Define the user and problem.
3. Propose the smallest useful version.
4. List what is explicitly out of scope.
5. Write acceptance criteria.
6. Identify dependencies, risks, and migration or rollout concerns.
7. Suggest tests, observability, and documentation updates.

## Output

Prefer:

- concise feature brief
- acceptance criteria
- delivery sequence
- tradeoff notes

## Planning checklist

- is there a smallest useful slice?
- are migration and rollback risks visible?
- are contracts or data changes called out explicitly?
- is human review required anywhere in the workflow?
