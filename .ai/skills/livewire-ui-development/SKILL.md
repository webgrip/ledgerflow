# Livewire UI Development Skill

## Use this when

- building a Livewire component
- creating a dashboard
- adding a form
- displaying structured or numeric data
- improving UX
- clarifying state-heavy operational workflows

## Principles

- clarity over cleverness
- visible loading/error states
- server-side validation
- authorization at the backend boundary
- no duplicated business logic in the UI
- state transitions should be understandable to operators

## State-heavy UI checklist

- units, signs, and totals are unambiguous
- intermediate vs final state is visible
- destructive or irreversible actions are confirmed
- timestamps and source/provider are visible where useful
- errors and retry states are understandable
- operator actions do not hide risk or finality
